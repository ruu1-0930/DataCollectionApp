# 阿里云迁移操作清单（阶段 0.5）

> 把现有系统迁到**自有的阿里云 ECS + RDS**。本文件是逐步可勾选的操作手册。
> 关联文档：[系统架构方案.md](系统架构方案.md) 阶段 0.5 与 FAQ 10.2、[CLAUDE.md](CLAUDE.md) 第 2 节。

**背景**：现服务器（后端阿里云 `8.156.78.198`、数据库腾讯云 `124.223.217.107`）均不在我方名下、随时可能停机；尚未开始采集，**无历史数据需迁移**，此刻迁移成本最低。

**目标架构**：`App → 阿里云 ECS(nginx + gunicorn + Flask) → 阿里云 RDS(MySQL，VPC 内网)`

---

## 准备：地域与 VPC（5 分钟）

- [ ] **选定地域**：所有资源（ECS、RDS）必须选**同一地域 + 同一可用区**。建议离团队近的，如「华东1(杭州)」。**定了不能改，后面全跟着它。**
- [ ] **VPC**：用默认 VPC 即可（新账号自带），记下网段（如 `172.16.0.0/12`）。

---

## 第 1 步：开通 RDS（MySQL）

控制台搜「RDS」→ 云数据库 RDS → 创建实例。

- [ ] 数据库类型：**MySQL 8.0**
- [ ] 系列：**高可用版**（省钱可先基础版/单可用区）
- [ ] 规格：**1核2G** 起步，存储 20G ESSD
- [ ] 付费：包月（或先按量）
- [ ] **网络：专有网络 VPC，选准备步骤的 VPC + 可用区**（关键：要与 ECS 同 VPC）
- [ ] 等实例变为 running

---

## 第 2 步：配置 RDS（建库 + 账号 + 白名单）

进入 RDS 实例详情页：

- [ ] **账户管理 → 创建账户**：用户名自定（**别再用 `lanya`**），授予 `lanya` 库读写。**记下新口令。**
- [ ] **数据库管理 → 创建数据库**：库名 `lanya`，字符集 **`utf8mb4`**
- [ ] **数据库连接 → 内网地址**：复制（形如 `rm-xxxx.mysql.rds.aliyuncs.com`）→ 这是后面 `.env` 的 `DB_HOST`
- [ ] **白名单设置**：先不要加公网。等第 4 步 ECS 建好，把 **ECS 内网 IP** 或 VPC 网段加入白名单

---

## 第 3 步：开通 ECS

控制台搜「ECS」→ 创建实例。

- [ ] **地域/可用区：与 RDS 完全一致**
- [ ] 规格：**2核4G**（`ecs.e-c1m2.large` 经济型 或 `ecs.g7` 通用型）
- [ ] 镜像：**Ubuntu 22.04 LTS 64位**
- [ ] 系统盘：40G ESSD
- [ ] **网络：同一个 VPC**；**分配公网 IPv4**，带宽固定 3Mbps（或按量）
- [ ] 登录：设 root 密码或绑定密钥对
- [ ] 记下：**公网 IP**（域名解析用）、**内网 IP**（回填 RDS 白名单用）
- [ ] 回到第 2 步白名单，加入 ECS 内网 IP

---

## 第 4 步：安全组（ECS）

ECS 实例 → 安全组 → 配置规则 → 入方向：

- [ ] 放行 **TCP 443**，源 `0.0.0.0/0`（HTTPS 对公网）
- [ ] 放行 **TCP 22**，源**仅你自己的公网 IP**（别对全网开）
- [ ] 放行 **TCP 80**，源 `0.0.0.0/0`（证书申请与 http→https 跳转）
- [ ] **不放行 5000**（gunicorn 仅本机回环，由 nginx 代理）
- [ ] 3306 不在 ECS 安全组开（走 RDS 内网）

---

## 第 5 步：域名 + HTTPS（建议）

> **服务器/备案决策（已定）**：**留在大陆节点 + 备案，不用海外节点。**
> - 海外节点虽免备案，但采集在上海、数据是**医疗个人敏感信息**，落到境外触发**数据出境安全评估/单独同意**（PIPL）+ 伦理/医院合规问题，得不偿失 → **排除海外节点**。
> - **免费试用 ECS 不能备案**：阿里云备案要求实例为**包年包月、时长 ≥ 3 个月、大陆地域**。→ 须把生产 ECS 买/转成「大陆 + 包月 ≥3 月」（**不是再买一台备用机**，是把现在的试用机换/续成合规机）。优先查现试用机能否直接转包月（可保留 IP，省去回填白名单/改地址/重打包）。
> - 关系链：医疗数据正式采集**必须 HTTPS** → HTTPS 几乎必须有**域名**（免费 DV 证书签域名，裸 IP 拿不到）→ 域名指向大陆机 → **必须备案**。
> - 测试阶段用 `http://IP` 不需要域名/备案，不卡进度；备案（约 1–2 周）可与 IP 测试并行。

- [ ] 把 ECS 换/续成 **大陆地域 + 包年包月 ≥3 个月**（备案前提）。若换实例致公网 IP 变更：回填 RDS 白名单、改三端 baseURL、重打包 App
- [ ] 有域名：**域名解析**把子域（如 `api.你的域名.com`）A 记录指向 ECS 公网 IP；提交**备案**（约 1–2 周）
- [ ] 无域名/等不及：先用**公网 IP + http** 跑通；但医疗数据红线要求 HTTPS，**正式采集前必须补上**
- [ ] **SSL 证书**：控制台「数字证书管理服务」→ 申请免费证书（DV，绑子域）→ 签发后下载 nginx 格式

---

## 第 6 步：ECS 上部署后端

SSH 登录 ECS（`ssh root@公网IP`）：

```bash
# 1. 基础环境
#    Ubuntu 22.04 默认 Python 3.10，但升级后的 numpy/scikit-learn 需 Python >= 3.11，故装 3.12
apt update && apt install -y software-properties-common nginx git
add-apt-repository -y ppa:deadsnakes/ppa
apt update && apt install -y python3.12 python3.12-venv
mkdir -p /opt/lanya && cd /opt/lanya
# 把 back/ 传上来（scp 或 git），假设到 /opt/lanya/back

# 2. 虚拟环境 + 依赖（务必用 python3.12 建 venv，不要用默认的 python3=3.10）
cd /opt/lanya/back
python3.12 -m venv venv && source venv/bin/activate
pip install -r requirements.txt gunicorn

# 3. 配 .env（填第 2 步的 RDS 内网地址、新口令）
cp .env.example .env
nano .env      # 填 DB_HOST/DB_USER/DB_PASS
python3 -c "import secrets; print(secrets.token_hex(32))"   # 生成 SECRET_KEY 填入 .env

# 4. 建表（schema 导入 RDS）
mysql -h <RDS内网地址> -u <新账号> -p lanya < database_schema_mysql.sql

# 5. 手动验证能起来
gunicorn -w 4 -b 127.0.0.1:5000 app:app
# 另开 ssh：curl http://127.0.0.1:5000 看响应，Ctrl+C 停掉
```

## 第 7 步：systemd 托管 gunicorn + nginx

> 配置文件已备好在仓库 `back/../deploy/`：`deploy/lanya.service`、`deploy/nginx-lanya.conf`。
> 上传：`scp -r deploy root@<ECS公网IP>:/opt/lanya/`

- [ ] systemd：`cp /opt/lanya/deploy/lanya.service /etc/systemd/system/` → `systemctl daemon-reload && systemctl enable --now lanya` → `systemctl status lanya`
- [ ] nginx：`cp /opt/lanya/deploy/nginx-lanya.conf /etc/nginx/sites-available/lanya` → 软链到 `sites-enabled`、删 `default` → `nginx -t && systemctl restart nginx`
- [ ] 验证：`curl -i http://127.0.0.1/` 返回 404 即代理通；外网测需安全组放行 **80**
- [x] **nginx 已切 HTTPS(443)**（2026-06-15）：`deploy/nginx-lanya.conf` 已是 443 版（域名 `api.sarcopenianus.com`、80→443 跳转）；证书 `/etc/nginx/ssl/api.sarcopenianus.com.{pem,key}`（key chmod 600）；`nginx -t` 通过、reload 生效
